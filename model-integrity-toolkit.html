What it is

The QDL Model Integrity Toolkit is a pre-verification structural screen for dimensional coherence and unit invariance. It is designed for internal governance and documentation defensibility, upstream of downstream processes.

Pre-verification Structural screen Document-based Domain-agnostic Internal use

Plain-English definitions

  • Dimensional admissibility: each equation is structurally consistent with declared units (e.g., unlike dimensions are not added).
  • Unit invariance: changing units (e.g., meters → feet) does not change the model’s meaning; the structure remains coherent under unit transformations.
  • Pre-verification: this screen runs before correctness testing; it checks whether the model is structurally coherent enough to verify.

The Toolkit reports structural status only (Present / Absent / Indeterminate) and stops.

What’s included

  • Dimensional Admissibility Checklist — a structured screen using Present / Absent / Indeterminate outcomes.
  • Pre-Verification Flow — an upstream gate: screen first, then proceed to QA/V&V outside the Toolkit.
  • Unit-Invariance Specification — definitions and structural criteria (no code; no thresholds).
  • Internal-Use License & Terms — bounded internal governance use; no support; no certification.

What it is not

  • Consulting or advisory services
  • Certification, endorsement, validation, or verification
  • Regulatory compliance determination
  • Guarantee of correctness or fitness for purpose
  • Implementation guidance, code, benchmarks, or tooling

The Toolkit is intentionally bounded: it reports structural status and stops.

How it fits

The Toolkit operates upstream of verification & validation (V&V), peer review, regulatory submission, and operational/decision use. It does not replace downstream processes; it prevents structurally incoherent models from reaching them.

Operational maturity

The QDL Model Integrity Toolkit is an internal-use document-based methodology. It does not require software integration. It can be applied using existing model artifacts (equations, unit tables, documentation, and code excerpts treated as documentation).

  • Included: checklist + unit-invariance specification + structured report format.
  • Not included: tooling, code, benchmarks, thresholds, or implementation guidance.

How to run it

The Toolkit is applied by collecting a small set of existing model artifacts and running a bounded structural screen. If required artifacts are missing, results are marked Indeterminate rather than inferred.

Minimum artifacts required

  • Model purpose and scope (2–5 sentences)
  • Inputs, parameters, and outputs (with units if declared)
  • Equations or logical relationships (as written)
  • Declared unit system(s) (SI / mixed / normalized / unknown)

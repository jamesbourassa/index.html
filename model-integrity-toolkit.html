What it is

The QDL Model Integrity Toolkit is a pre-verification structural screen for dimensional coherence and unit invariance. It is designed for internal governance and documentation defensibility, upstream of downstream processes.

Pre-verification Structural screen Document-based Domain-agnostic Internal use

What’s included

  • Dimensional Admissibility Checklist — Present / Absent / Indeterminate screening with explicit outcomes.
  • Model Pre-Verification Flowchart — placement upstream of QA, V&V, audit, or operational use.
  • Unit-Invariance Test Specification — formal definitions, allowed/prohibited transformations, and structural criteria.
  • Internal-Use License & Terms — non-exclusive internal use, no support, no certification, no guarantees.

What it is not

  • Not consulting or advisory services.
  • Not certification, endorsement, validation, or verification.
  • Not a regulatory compliance determination.
  • Not a guarantee of correctness, accuracy, or fitness for purpose.
  • Not implementation guidance, tooling, code, benchmarks, or threshold tuning.

The Toolkit is intentionally bounded: it reports structural status and stops.

How it fits

The Toolkit is designed to operate before verification & validation (V&V), peer review, regulatory submission, or operational/decision use. It does not replace downstream processes; it prevents structurally incoherent models from reaching them.

How to run it

The Toolkit includes an LLM-ready runner prompt (“MIT Runner”) that executes a bounded screen and generates a report. A valid run requires only four artifacts.

Minimum artifacts required

  • Model purpose and scope (2–5 sentences)
  • Inputs, parameters, outputs (names; units if declared)
  • Equations or logical relationships (as written)
  • Declared unit system(s) (SI / mixed / normalized / unknown)

Operator entry point

/ONE_SHOT_MIT_SCREEN

Quick demo sample (domain-agnostic)

Model purpose and scope: This model estimates an abstract system response based on three input quantities and one scaling parameter. Inputs, parameters, outputs: Input A — unit: U1 Input B — unit: U2 Input C — unit: (not specified) Parameter K — unit: U1/U2 Output R — unit: U1 Equations / relationships: R = K × (A + B) / C Declared unit system: Mixed units; no specific system declared.

Note: the Toolkit uses audit language only (Present / Absent / Indeterminate; Pass / Fail / Indeterminate) and stops after the report.
